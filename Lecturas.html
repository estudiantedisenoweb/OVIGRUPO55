Documentacion De Pruebas De Software



Siendo los softwares una herramienta que se utilizan en prácticamente en todo el mundo y funciona con todos los dispositivos se debe de garantizar el buen funcionamiento de este y que sea un software de calidad para el buen consumo.


wordpress. (07 de 08 de 2010). Obtenido de https://dy05.wordpress.com/


Durante este proceso los desarrolladores suelen olvidar de un paso muy importante que es la documentación, ¿esto para qué? Por los siguientes motivos.

Para un mantenimiento efectivo por parte de sus desarrolladores o personal ajeno a este.

para que el usuario tenga un buen manejo y saque provecho de este, hay muchos usuarios que no entiende en su totalidad un software y mucho menos lo manejan, pero es por falta de información.

Para verificar antes de instalar los requisitos del software y así evitar un mal funcionamiento del equipo o del software.



Tecnicas De Pruebas De Software



Existen varias técnicas de software, pero unas de las más comunes que se realizan son las técnicas de caja blanca (white box testing) y la técnica de caja negra (black box testing).

La técnica de caja blanca: son las que realiza los desarrolladores con el fin de validar el código paso a paso con esto aseguramos una muy buena calidad en el software.

Jose Alfonso . (11 de 08 de 2013). blogspot. Obtenido de http://taller1cdsucn.blogspot.com/2013/08/tecnicas-de-caja-negra.html


Técnicas de caja negra: las realiza los equipos de testing validando todas las entradas y salidas del software sin tener en cuenta el funcionamiento interno, esto nos asegura que los datos de entrada correspondan a los datos de salida.

Guevara , O. (15 de 05 de 2011). blogspo. Obtenido de http://nightcrowl.blogspot.com/2011/05/pruebas-unitarias.html
